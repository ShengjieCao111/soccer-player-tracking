# 足球球员多目标跟踪与运动分析

## 项目简介
本项目为计算机视觉课程实训项目，基于 Python 与 OpenCV 实现足球视频中球员的多目标跟踪与运动数据自动分析。
项目实现了**球员跟踪、轨迹提取、速度统计、中线越界检测、运动热力图可视化**五大核心功能，完整覆盖了从视频输入到运动分析结果输出的全流程，是计算机视觉技术在体育分析领域的轻量化应用实践。

---

## 项目背景与意义
随着计算机视觉技术的发展，体育视频自动分析成为重要应用方向。传统人工统计球员跑动数据效率低、误差大，而本项目通过自动跟踪与分析，可快速获取球员跑动速度、活动范围、越界时间等关键数据，为战术分析提供基础支持。
本次实训以足球比赛视频为对象，通过传统跟踪算法实现球员运动分析，帮助理解视频目标跟踪、运动轨迹分析、数据可视化的完整流程。

核心功能模块与实现过程
1. 视频读取与预处理模块
功能：读取视频文件，获取视频帧率、尺寸，计算球场中线位置，为后续处理提供基础。
实现过程：
使用 cv2.VideoCapture 读取视频文件；
通过 cap.get() 获取视频帧率（FPS）与画面宽高；
计算画面中线横坐标 mid_line = 视频宽度 // 2，作为越界检测基准；
定义 processImg() 函数，实时计算并显示帧率，优化播放体验。
2. 多目标跟踪模块
功能：支持手动框选多名球员，使用 OpenCV 传统跟踪算法实现稳定跟踪，实时显示跟踪框与球员编号。
实现过程：
定义跟踪算法字典，封装 CSRT、KCF、MOSSE 等 OpenCV 内置跟踪器；
使用 cv2.selectROI() 手动框选球员，将框选结果存入列表；
初始化 cv2.legacy.MultiTracker，为每个球员创建独立跟踪器并添加；
在主循环中调用 trackers.update() 实时更新球员位置，绘制绿色跟踪框与球员中心点。
3. 轨迹提取与速度分析模块
功能：提取球员运动轨迹，计算实时速度并统计平均速度、最大速度。
实现过程：
从跟踪框中提取球员中心点坐标 (cx, cy)，按帧号存储轨迹数据；
定义像素 - 实际距离换算比例 PIXEL_TO_M = 0.02（即 1 像素对应 0.02 米）；
计算相邻帧像素距离，换算为实际距离；
结合视频帧率计算时间差，通过 速度 = 实际距离 / 时间差 得到实时速度；
视频结束后，对速度数据求均值与最大值，统计球员全程运动数据。
4. 中线越界检测模块
功能：实时判断球员是否越过球场中线，记录首次越界时间并输出。
实现过程：
在视频画面中绘制红色中线，直观显示越界基准；
初始化每个球员的初始侧别（中线左侧 / 右侧）；
逐帧判断球员中心点侧别，若与初始侧别不同且未记录越界行为，则判定为越界；
记录越界帧号，结合视频帧率换算为具体时间，控制台输出球员 ID 与越界时间。
5. 运动热力图可视化模块
功能：基于球员轨迹生成运动热力图，叠加球场背景展示球员活动区域。
实现过程：
创建与球场背景尺寸一致的空白矩阵；
使用 cv2.line() 绘制球员运动轨迹；
对轨迹矩阵进行高斯模糊处理，生成平滑热区；
归一化处理后，通过 cv2.applyColorMap() 生成彩色热力图；
使用 cv2.addWeighted() 将热力图与球场背景叠加，调整透明度优化显示效果。

项目难点与解决方法
跟踪稳定性问题：部分算法在球员快速运动或镜头晃动时容易丢失目标，通过对比测试，最终选用 CSRT 算法，其对快速运动与视角变化的适应性更强，跟踪稳定性显著提升。
速度计算误差问题：相邻帧跟踪框微小抖动会导致速度数值波动，通过增加分母防零处理，并对速度数据进行统计分析，确保结果准确。
越界误判问题：未设置状态标记时，同一球员多次触发越界提示，通过 cross_flag 标记球员越界状态，仅记录首次越界行为，避免误判。
热力图显示效果差问题：初始生成的热力图模糊或层次不明显，通过调整高斯模糊核大小与叠加透明度，优化了热力图的清晰度与可读性。
项目不足与改进方向
现有不足
需手动框选球员，无法自动识别目标；
实时速度存在小幅波动，未做平滑处理；
仅支持单人热力图显示，无法对比多名球员活动区域；
数据结果仅在控制台输出，未保存为文件格式。
改进方向
引入目标检测算法，实现球员自动识别与跟踪；
增加速度平滑处理，如滑动平均，减少数值波动；
优化热力图功能，支持多名球员热力图叠加对比；
添加数据保存模块，将速度、越界信息导出为 Excel 或 CSV 文件；
优化界面交互，增加视频暂停、重新跟踪等功能。
